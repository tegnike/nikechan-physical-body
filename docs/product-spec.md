# Product Spec

## コンセプト

AIニケちゃんの物理ボディは、走行ロボットではなく「卓上/部屋内の物理アシスタント」として設計する。

主役は会話、通知、状態表示、作業支援。移動とグリッパーは、実用機能を補強する身体性・注意喚起・演出として扱う。

## 目標

- マスターが声で呼べば返事する。
- 画面の顔と表情で、待機中/考え中/通知あり/エラー/会話中が分かる。
- 作業中に必要な通知、タイマー、タスク、Discord/xangi関連の状態を知らせる。
- 安全な範囲で短距離移動や身振りを行い、存在感を出す。
- カメラで周囲確認や簡易状態確認ができる。
- CoreS3 SEをサブコンソールとして使い、メインボディを止める/起こす/状態確認できる。

## 非目標

- 部屋全体を自律巡回すること。
- 人を追跡すること。
- 物を確実につかんで運ぶこと。
- 障害物だらけの環境で安全に自走すること。
- CoreS3単体でLLM/音声認識/音声合成モデルを動かすこと。

これらはセンサー・電源・機構が増えてから検討する。

## MVP

### 会話

- ウェイクワード: `ニケちゃん`
- ウェイクワードON時は、音量候補検出後に短い音声をbridgeへ送り、呼ばれた時だけ本会話へ進む。
- 本会話は無音判定で録音終了する。
- 返答はAivis Cloud音声で再生する。
- 返答中は移動しない。

### 表情

最低限の状態:

- idle: 待機
- listening: 聞き取り中
- thinking: 返答生成中
- speaking: 発話中
- alert: 通知あり
- error: 接続/API/ロボット制御エラー

### 通知

MVPではbridge側から以下を読み上げ可能にする。

- 手動テキスト通知
- タイマー終了
- エラー通知

次段階でxangi/Discord/タスク連携を追加する。

### 移動

安全制約:

- 音声コマンドで移動する場合は最大1秒、低速のみ。
- 障害物センサー追加までは自律移動を主機能にしない。
- 会話中、録音中、発話中は停止。
- `stop` は常に最優先。

主なモーション:

- 軽く前後する
- 左右に小さく振る
- 旋回で反応する
- グリッパーで手振り風の動き

## 操作モデル

### 通常

1. マスターが「ニケちゃん」と呼ぶ。
2. 本体が発話を録音する。
3. bridgeが返答を作る。
4. 本体が返答を喋る。
5. 必要なら短い身振りをする。

### 手動

- CoreS3本体の顔タップ: 会話開始
- 本体Web UI: 設定、手動移動、カメラ、会話開始
- CoreS3 SE: 将来のサブコンソール

## 追加センサー方針

優先度順:

1. 前方ToF距離センサー
   - ぶつかる前に止まる
   - 短距離移動の安全性向上
2. 底面落下検知
   - 机上運用するなら重要
3. バッテリー/電源監視
   - 長時間運用の安定化
4. 広角カメラまたは外部カメラ
   - 周囲認識を強化する場合

## CoreS3 SEの役割案

MVP後に、SEを「サブコンソール」として実装する。

画面:

- メインボディ接続状態
- bridge状態
- マイク/スピーカー状態
- ウェイクワード状態
- 直近の発話/通知

操作:

- 緊急停止
- 起こす/寝かす
- ウェイクワードON/OFF
- カメラ表示切替
- 手動リモコン

## 成功条件

MVP成功:

- 呼びかけで反応する。
- 返答音声が自然に返る。
- 状態が画面で分かる。
- 手動停止が確実に効く。
- 低速の身振りが安全に動く。

次段階成功:

- 通知/タスクが実用的に届く。
- SEサブコンソールから停止・状態確認ができる。
- 前方距離センサーで衝突前停止できる。
